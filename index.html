რა არის React?

React — JavaScript-ის ბიბლიოთეკა მომხმარებლის ინტერფეისის ასაგებად: კოდით აღწერ, როგორი უნდა იყოს UI მოცემული props/state-ის მიხედვით. კომპონენტები მრავალჯერ გამოყენებადია; React ასინქრონებს რეალურ DOM-ს.

Virtual DOM და რენდერი

React ინახავს UI-ის ვირტუალურ წარმოდგენას. ცვლილებისას აითვლება „უაღრესად მსუყე“ განსახიერება და ბრაუზერის DOM-ს მხოლოდ საჭირო განახლებები ერთიანდება — ეს ამცირებს პირდაპირ მანიპულაციებს და ხელს უწყობს წინასწარმეტყველებადობას.

ერთმიმართულიანი მონაცემთა ნაკადი

მონაცემები ჩვეულებრივ ზემოდან ქვემოთ მიედინება (props). შვილისგან ასლის მიღება callback-ებით ან state lift-ით ხდება. ეს მოდელი ხსნის, ვინ წარმოადგენს სიმართლეს და ამარტივებს დებაგს.

Hooks და ფუნქციური კომპონენტები

Hooks (მაგ. useState, useEffect) საშუალებას იძლევა მდგომარეობა და გვერდითი ეფექტები ფუნქციაში იყოს ლოკალიზებული, კლასების გარეშე. წესი: Hooks მხოლოდ კომპონენტის ტოპ ლეველზე, პირობითი ხაზების გარეშე.

SPA და რატომ სჭირდება Router

ერთგვერდიან აპში ნავიგაცია URL-ს იყენებს გვერდის მდგომარეობის სინქრონიზაციისთვის. React Router აკავშირებს URL-ს კომპონენტებთან და ბრაუზერის ისტორიას.

გლობალური state და Redux Toolkit

ლოკალური state კომპონენტში რჩება; საერთო მონაცემები (მაგ. კალათა, იუზერი) ხშირად ცენტრალიზებულ store-ში მოდის. Redux Toolkit slice-ებსა და reducers-ს ამარტივებს; Immer საშუალებას იძლევა, კოდში ჩანდეს პირდაპირი მინიჭება, რეალურად კი უსაფრთხო განახლება მიმდინარეობდეს.

კონტროლირებადი ფორმები და ვალიდაცია

ფორმის მნიშვნელობა React state-ში ინახება — ე.წ. controlled components. Formik აერთიანებს მაჩენერებელს; Yup ადგენს წესებს შეცდომების ტექსტებით.

თეორიული ფონი

Hooks მოჰყვება გამოძახების თანმიმდევრობას ყოველ რენდერზე. ამიტომ არ შეიძლება ისინი იყოს ციკლში ან პირობით ბლოკში — წყობა დაირღვევა. Side effect-ები useEffect-ში იჭერება, რათა რენდერი რჩებოდა „სუფთა“.

useState — მდგომარეობის მართვა
import { useState } from 'react';

function Counter() {
  // [მნიშვნელობა, განახლების ფუნქცია] = useState(საწყისი_მნიშვნელობა)
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>რაოდენობა: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
    </div>
  );
}
useEffect — გვერდითი ეფექტები
import { useState, useEffect } from 'react';

function Timer() {
  const [time, setTime] = useState(0);

  useEffect(() => {
    // კომპონენტის mount-ზე გაეშვება
    const interval = setInterval(() => {
      setTime(t => t + 1);
    }, 1000);

    // cleanup — unmount-ზე
    return () => clearInterval(interval);
  }, []); // [] = მხოლოდ ერთხელ, mount-ზე

  return <p>{time} წამი</p>;
}
useEffect-ის dependency array
useEffect(fn, [])
მხოლოდ mount-ზე ერთხელ. componentDidMount-ის ეკვივალენტი.
useEffect(fn, [val])
ყოველ ჯერ, როცა val შეიცვლება. componentDidUpdate-ის ეკვივალენტი.
useMemo და useCallback — ოპტიმიზაცია
// useMemo — ძვირი გამოთვლების ქეშირება
const result = useMemo(() => {
  return heavyCalculation(data);
}, [data]); // მხოლოდ data-ს ცვლილებაზე გადაითვლება

// useCallback — ფუნქციის მემოიზაცია
const handleClick = useCallback(() => {
  doSomething(id);
}, [id]); // ახალი ფუნქცია მხოლოდ id-ს ცვლილებაზე

Redux მიჰყვება ერთმიმართულიან მონაცემთა ნაკადს: Action → Reducer → Store → View. ცენტრალიზებული store დიდ აპებში დებაგს ამარტივებს; Redux Toolkit ამცირებს boilerplate-ს და იყენებს Immer-ს draft state-ის უსაფრთხო ცვლილებებისთვის.

ინსტალაცია
npm install @reduxjs/toolkit react-redux
createSlice — State ლოგიკა
// features/counterSlice.js
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => { state.value += 1; },
    decrement: (state) => { state.value -= 1; },
    setAmount: (state, action) => {
      state.value = action.payload;
    },
  },
});

export const { increment, decrement, setAmount } = counterSlice.actions;
export default counterSlice.reducer;
configureStore — Store-ის შექმნა
// store/index.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from '../features/counterSlice';

export const store = configureStore({
  reducer: {
    counter: counterReducer,
  },
});
Provider და კომპონენტებში გამოყენება
// main.jsx
import { Provider } from 'react-redux';
import { store } from './store';

ReactDOM.createRoot(document.getElementById('root')).render(
  <Provider store={store}>
    <App />
  </Provider>
);

// კომპონენტში
import { useSelector, useDispatch } from 'react-redux';
import { increment } from './counterSlice';

function CounterDisplay() {
  const count = useSelector((state) => state.counter.value);
  const dispatch = useDispatch();

  return <button onClick={() => dispatch(increment())}>{count}</button>;
}

კონტროლირებადი ელემენტებისას მნიშვნელობა React state-შია — ერთი წყარო სიმართლისა. Formik აერთიანებს შეხებას, შეცდომებსა და submit-ს; Yup სქემით შედარებადი წესები იჭრება დეკლარაციულად, ხშირად სინქრონულ ვალიდაციასთან.

ინსტალაცია
npm install formik yup
Yup — ვალიდაციის სქემა
import * as Yup from 'yup';

const validationSchema = Yup.object({
  name: Yup.string()
    .required('სახელი სავალდებულოა')
    .min(2, 'მინ. 2 სიმბოლო'),

  email: Yup.string()
    .email('ელ-ფოსტა არასწორია')
    .required('ელ-ფოსტა სავალდებულოა'),

  age: Yup.number()
    .min(18, '18 წლიდან')
    .max(100, 'მაქს. 100')
    .required('ასაკი სავალდებულოა'),
});
useFormik Hook
import { useFormik } from 'formik';

function RegisterForm() {
  const formik = useFormik({
    initialValues: { name: '', email: '', age: '' },
    validationSchema,
    onSubmit: (values) => {
      console.log('გაგზავნა:', values);
    },
  });

  return (
    <div onSubmit={formik.handleSubmit}>
      <input
        name="name"
        value={formik.values.name}
        onChange={formik.handleChange}
        onBlur={formik.handleBlur}
      />
      {formik.touched.name && formik.errors.name && (
        <span className="error">{formik.errors.name}</span>
      )}
      <button type="submit">რეგისტრაცია</button>
    </div>
  );
}
